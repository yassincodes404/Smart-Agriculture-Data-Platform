## What We Built Today

Today we built the **User and Authentication system** for the backend. This means the platform now has working login, registration, and user management features accessible through HTTP API endpoints.

---

## 1. The Goal

Before today, the backend had only two working endpoints:
- `GET /api/health` — checks if the backend is running
- `GET /api/health/db` — checks if the database is connected

Everything else was empty placeholder files. The goal today was to add:

```
POST /api/v1/auth/register   → create a new account
POST /api/v1/auth/login      → log in and receive a token
GET  /api/v1/auth/me         → see your own profile
POST /api/v1/auth/logout     → log out

GET    /api/v1/users/        → list all users (admin only)
GET    /api/v1/users/{id}    → see a user by ID (admin only)
PUT    /api/v1/users/{id}    → edit a user (admin only)
DELETE /api/v1/users/{id}    → delete a user (admin only)
```

## 3. How Login Works (Simple Explanation)

1. User sends their email and password to `POST /api/v1/auth/login`
2. The backend finds the user in the database by email
3. The backend checks if the password matches (using secure hashing — we never store plain passwords)
4. If correct, we generate a **JWT token** (a signed string that proves who you are)
5. The user sends this token with every future request
6. Protected endpoints verify the token before responding

## 6. Does This Affect the Real Database?

**No.** The tests are completely isolated from MySQL.

- Tests use a **SQLite database in memory** (a tiny temporary database that lives only while tests run)
- The **real MySQL database is never contacted** during testing
- Each test starts with a fresh empty database and cleans up after itself
- When running in Docker for real, the MySQL connection works exactly as before

The change we made to `db/session.py` (lazy connection) means:
- **Before:** MySQL engine was created the moment Python imported the file
- **After:** MySQL engine is only created when the first real request comes in

This makes zero difference to production behavior — MySQL still connects and works normally when Docker is running.
