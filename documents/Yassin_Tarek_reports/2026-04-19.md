## 1. What we accomplished

### 1.1 Per-land climate (answer: yes, now supported)

- After **`POST /api/v1/lands/discover`**, the discovery pipeline calls **Open-Meteo** (`forecast` API, current block) using the landÔÇÖs **latitude and longitude**.
- On success it inserts a row into **`land_climate`** and links provenance to **`data_sources`** (`Open-Meteo`).
- If the external API fails, the land is still marked **`active`** and the error is logged (no hard failure of onboarding).
- **`GET /api/v1/lands/{land_id}/timeseries?metric=climate`** returns stored points: `timestamp`, `value` (temperature ┬░C), `payload` (`humidity_pct`, `rainfall_mm`).
